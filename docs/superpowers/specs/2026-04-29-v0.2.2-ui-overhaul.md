# v0.2.2 UI/UX Overhaul Plan

## Context

v0.2.0 shipped rich-text, drag reorder, and puppeteer PDF — but the UI is visually bare compared to professional resume builders. User provided two reference screenshots showing: (1) dashboard with tall A4-ratio resume preview cards on a dark background, (2) editor with collapsible icon-decorated sections, independent panel scroll, and polished toolbar. This release closes the visual quality gap without adding new features.

## Decisions (locked in)

- **Photo storage:** Vercel Blob (`@vercel/blob`, 1GB free tier)
- **Drag library:** Replace dnd-kit → `@atlaskit/pragmatic-drag-and-drop`
- **Dashboard preview:** Layout component scaled-down rendering
- **Dark mode:** `next-themes` ThemeProvider + toggle (already installed, not wired)
- **Icons:** lucide-react everywhere (replace emoji in Modern template)

### T4: Replace dnd-kit → @atlaskit/pragmatic-drag-and-drop (🔴 highest risk)

### T5: Fix independent panel scroll
- In `editor-client.tsx` desktop grid: change `min-h-[calc(100vh-3.5rem-4rem)]` → `h-[calc(100vh-3.5rem-4rem)]`

- Mobile: no change needed (tabs already isolate content)

## Key risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| @atlaskit/pragmatic-drag-and-drop API unfamiliar | Drag breaks entirely | Do T4 first, test manually before proceeding |
| Vercel Blob needs BLOB_READ_WRITE_TOKEN | Photo upload 500s | Add to .env.example + document; graceful error if missing |
| Template dark mode leak | Resume preview gets dark bg | Explicit `bg-white text-black` wrapper on all template content |
| Dashboard preview N×Layout renders | Slow with 10+ resumes | Layout is pure RSC with no hooks; acceptable at ≤100 users |
